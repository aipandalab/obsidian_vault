### 00:02:39 - 00:07:00 | 硬件需求：显卡（Nvidia Cuda vs AMD ROCm）与显存

幸运的是，较新的 AMD Radeon 显卡也有了能够与 Nvidia CUDA 竞争的技术，叫做 ROCm（Radeon Open Compute，Radeon 开放计算），而现在 Ollama 已经完全支持它了，这真的很棒。你可以查看 Ollama 在 GitHub 仓库里的官方文档页面，这里有一个表格展示了哪些制造商的哪些显卡可以与 Ollama 协同工作。

你之所以需要关心显存（VRAM）的大小，是因为模型越大，你需要用来将其加载到内存中的显存就越多。例如，在 Meta 的文档页面上可以看到，较小的 7B（70 亿参数）Llama 模型至少需要 8 GB 的显存；13B 模型需要 16 GB；而对于 70B 模型，你将需要 64 GB 的显存，这在目前已经超出了我这个项目的范围。我当时的希望是，凭借 Radeon 7900 XT，我将能够同时加载一个 13B 模型加上一个较小的 7B 模型。你会在我后面的测试中看到，这完全可行。

一旦我创建了 LXC 容器，接下来同样重要的是直通 AMD 显卡。为此我需要找到另一个变通方法，因为它与虚拟机中的 PCI 直通有点不同。你必须给它一个设备路径（Device Path），而不是从下拉菜单中直接选择显卡。AMD 显卡直通需要两个设备路径：`/dev/kfd` 和 `/dev/dri`。第一个成功了，但对于第二个，我不幸遇到了报错。

幸运的是，我在另一个 Proxmox 论坛帖子里找到了解决方案。我需要进入 Proxmox 的命令行（CLI），修改该 LXC 容器的配置文件（`.conf`），加入以下这四行代码：

Plaintext

```
lxc.cgroup2.devices.allow: c 226:* rwm
lxc.cgroup2.devices.allow: c 511:* rwm
lxc.mount.entry: /dev/crypto/kfd dev/crypto/kfd none bind,optional,create=dir
lxc.mount.entry: /dev/dri dev/dri none bind,optional,create=dir
```

这些行确保了 AMD 显卡在启动时能正确直通到 LXC 容器中。

完成之后，我登录到 LXC 容器中，做了一些基础的设置工作，比如更新 APT 软件包、添加一个管理员用户、将我的 SSH 密钥复制给该用户，然后安装了 AMD 驱动和 ROCm 驱动。我在这里就不详细介绍所有这些步骤了。我想在未来的某个时候，我需要专门制作一个关于 Proxmox 中 LXC 容器的详细教程，我想有些人在以前的视频里也向我提过这个要求。至于 AMD 驱动的部分，我基本上只是遵循了 AMD 官方文档中描述的步骤进行复制粘贴，我会把我在视频中使用的文档页面链接放在描述里。
